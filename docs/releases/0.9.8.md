# Videorc 0.9.8 Beta 1

Videorc 0.9.8 is the **permissions-only onboarding** release: first launch
now exists solely to collect macOS grants (and skips itself when they
already exist), plus 4K recording unlocked on every tier and the round
camera-bubble fix.

## Scope

- **Permissions-only onboarding** (PO1–PO4) — the 4-step first-run tour
  (workflow chooser that wrote captureConfig, setup checklist, privacy
  checkbox, finish picker) is deleted. Onboarding is one dialog: the three
  System access rows (shared `systemAccessRows()` derivation) with the
  grant action each permission supports — camera/mic fire the native macOS
  prompt in place via the new `system:request-media-access` IPC (a grant
  restarts the capture backend, mirroring the grant watcher), Screen
  Recording deep-links System Settings. The gate is permission state, not a
  version flag: evaluated once per launch after the backend connects and
  device enumeration arrives; the localStorage key survives as an any-value
  snooze (existing installs and seeded probes stay suppressed). Settings →
  System access gains "Set up permissions" (force-open). Acceptance:
  `docs/acceptance/2026-07-06-permissions-onboarding.md`.
- **4K recording free on every tier** — the recording entitlement no longer
  gates 4K; 4K streaming stays Premium.
- **Circle camera mask** — the CPU compositor's mask stretched to the dest
  box (ellipse when the preview drawable's aspect drifted from the
  output's); CPU blit and Metal shader now inscribe a true circle
  (min(w,h), centered), matching FFmpeg's `circle_alpha_mask_filter`;
  regression tests in both compositors.

## Notes

- Fresh installs now see the permissions dialog instead of the tour; users
  whose Mac already holds the grants (or who updated in place) see nothing.
- The permissions dialog never auto-runs the mic probe — the mic chip reads
  "Checked on first use" until the user clicks Enable or runs a mic check.
